Before writing any code, enforce the four discipline rules:

1. **Think Before Coding** — state assumptions explicitly, surface ambiguity, push back on over-engineering before touching a file
2. **Simplicity First** — minimum code that solves the problem; no speculative features, abstractions, or configurability
3. **Surgical Changes** — touch only what was asked; never "improve" adjacent code; every diff line must trace to the request
4. **Goal-Driven Execution** — define verifiable success criteria upfront, state a brief plan, loop until verified

Do NOT start implementing until all four rules have been applied to the current task.
